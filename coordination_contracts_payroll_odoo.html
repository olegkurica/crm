Модуль координації в Odoo: контракти, табелі годин, виплати зарплат

Відділ координації — Odoo

Контракти, табелі, виплати

Бригадир → Координатор → Головний координатор

Роль 1

Бригадир

Веде 1 контракт. Щодня вносить години кожного працівника, фіксує завдатки, покарання, доплати. Бачить тільки свій контракт.

1 контракт Табель годин Завдатки/штрафи
Роль 2

Координатор

Веде кілька контрактів. Бачить табелі всіх своїх бригадирів, перевіряє і затверджує місячні виплати. Бачить тільки свої контракти.

N контрактів Перевірка табелів Затвердження виплат
Роль 3

Головний координатор

Бачить усі контракти всіх координаторів. Зведений дашборд по виплатах, перегляд і виправлення будь-якого табеля, звіти по місяцям.

Всі контракти Зведений звіт Повний доступ
Модель данихЯк пов'язані об'єкти в Odoo
1

Контракт (project.project або custom)

Назва клієнта, дата початку/кінця, ставка за годину, валюта. Прив'язаний до компанії-клієнта в CRM. Поле "Координатор" і "Бригадир".

2

Список працівників на контракті

Many2many зв'язок: контракт ↔ список hr.employee. Кожен найнятий кандидат прив'язується до конкретного контракту.

3

Табель (account.analytic.line)

Один запис = 1 день × 1 працівник × кількість годин. Поля: дата, працівник, контракт, години, нотатка бригадира.

4

Фінансові коригування

Окрема модель або additional_line в табелі: тип (завдаток / штраф / доплата), сума, дата, коментар, хто вніс.

5

Місячний розрахунок зарплати

hr.payslip або custom: сума = (години × ставка) + доплати − завдатки − штрафи. Статус: Чернетка → Перевірено → Затверджено → Виплачено.

6

Архів виплат

Після затвердження — заблокований запис. Зміни тільки через коригування. Повна трасованість хто/коли/що змінив.

Це макет того, як виглядає картка контракту в Odoo після налаштування. Один контракт = один клієнт + список людей + бригадир.

ПроблемаЧому Excel не сходиться

Ручні помилки при копіюванні

Формули ламаються при додаванні рядків. Одна людина редагує — інша не бачить зміни.

Завдатки в окремому файлі

Бригадир записав завдаток — бухгалтер не знає. Не підтягується автоматично в підсумок.

Немає статусів і затвердження

Незрозуміло — виплачено чи ні. Немає підпису "затверджено". Можна змінити заднім числом.

Версії файлів

"Фінальний_v3_справжній_ОНОВЛЕНО.xlsx" — класична ситуація. Немає єдиного джерела правди.

План переносуExcel → Odoo покроково
1

Підготувати Excel для імпорту

Стандартизувати колонки: ПІБ, контракт, місяць, дата, тип запису (години / завдаток / штраф / доплата), сума/кількість. Один рядок = один запис.

2

Визначити глибину архіву

Скільки місяців назад переносимо? Рекомендація: останні 3–6 місяців активних записів. Старіші — архівувати в PDF або окремому файлі.

3

Спочатку — список працівників і контрактів

Імпортувати hr.employee та контракти. Тільки після цього — прив'язані до них записи годин і коригувань.

4

Імпортувати табелі (account.analytic.line)

CSV через Odoo Import. Поля: employee_id, date, unit_amount (години), project_id (контракт), name (нотатка).

5

Імпортувати фінансові коригування

Окрема кастомна модель (через Studio). Тип, сума, дата, працівник, контракт — окремий CSV.

6

Звірка: Odoo vs Excel

Розрахувати підсумки за останній закритий місяць в обох системах. Розбіжності — виправити до go-live. Це критичний крок.

7

Freeze Excel і перехід

З 1-го числа наступного місяця — тільки Odoo. Excel заморожується, зберігається як read-only архів.

Обов'язкові модулі

Timesheets (hr_timesheet)

Базовий модуль для внесення годин. Бригадир входить і бачить форму по своєму контракту (project).

Денний табельАналітика годин

Project (project)

Контракт = Project. Дає Kanban, список завдань, права доступу по проєктах. Бригадир — "project user" тільки свого проєкту.

Контроль доступу

Payroll (hr_payroll)

Розрахунок зарплати на основі табеля. Payslip з формулами: base + rules (доплати/штрафи). Статуси: Draft → Confirm → Done.

Salary rulesPayslip

Odoo Studio

Без коду: додати кастомні поля (завдаток, штраф, тип коригування) до контракту і payslip. Потрібна Enterprise ліцензія.

Automated Actions — нагадування
1

Нагадування бригадиру

Щопонеділка: "Ви не внесли години за минулий тиждень по контракту X". Умова: якщо в понеділок ранок немає записів за попередні 5 днів.

2

Нагадування координатору

1-го числа кожного місяця: "Перевірте та затвердіть табель за попередній місяць по контрактах [список]".

3

Алерт по невідповідності

Якщо кількість годин на тиждень по працівнику = 0, а статус "активний" — автоматичний alert координатору.
